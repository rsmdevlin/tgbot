# 🏢 AI-Офис v2 — Telegram Bot

Бот для управления командой AI-агентов прямо в Telegram.
Агенты-инженеры могут **читать и патчить код бота на GitHub** — Railway автоматически деплоит изменения.

## ⚡ Быстрый старт

### 1. Создай репо на GitHub

Залей в него эти файлы:
```
main.py
requirements.txt
railway.toml
```

### 2. Подключи Railway к GitHub

1. Зайди на [railway.app](https://railway.app)
2. New Project → Deploy from GitHub repo → выбери своё репо
3. Railway автоматически задеплоит бота

### 3. Настрой переменные в Railway

В разделе **Variables** добавь:

| Переменная | Описание | Пример |
|---|---|---|
| `TELEGRAM_TOKEN` | Токен от @BotFather | `7123456789:AAF...` |
| `GROQ_API_KEY` | Ключ с [console.groq.com](https://console.groq.com) | `gsk_...` |
| `OWNER_ID` | Твой Telegram ID (узнай у @userinfobot) | `6091955295` |
| `GITHUB_TOKEN` | Personal Access Token с правом `repo` | `ghp_...` |
| `GITHUB_REPO` | Репозиторий | `username/ai-office-bot` |
| `GITHUB_BRANCH` | Ветка | `main` |
| `GITHUB_FILE` | Путь к файлу бота в репо | `main.py` |
| `GROUP_CHAT_ID` | ID группового чата (опционально) | `-1001234567890` |

### 4. Получи GitHub Token

1. GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. Generate new token → отметь `repo` (полный доступ к репо)
3. Скопируй токен → вставь в Railway как `GITHUB_TOKEN`

### 5. Узнай GROUP_CHAT_ID (опционально)

1. Создай группу в Telegram
2. Добавь своего бота в группу
3. Напиши `/id` в группе
4. Скопируй ID в Railway Variables

## 🤖 Как работает self-патчинг

```
Ты пишешь агенту-инженеру задачу
        ↓
Инженер читает текущий код из GitHub
        ↓
Groq LLaMA генерирует исправленный код
        ↓
Ты нажимаешь "✅ Применить патч"
        ↓
Бот коммитит код через GitHub API
        ↓
Railway видит пуш → автодеплой
        ↓
Бот перезапускается с новым кодом (~1-2 мин)
```

## 🔧 Создание агента-инженера

При добавлении агента используй роль вроде:
- `"Инженер-разработчик бота"`
- `"Python developer"`
- `"Программист"`

Бот автоматически определит его как инженера и даст доступ к коду.

## 📋 Функции бота

- **👥 Команда** — список всех агентов
- **➕ Агент** — добавить нового агента
- **📋 Задачник** — трекер задач с историей
- **💬 Чат** — диалог с конкретным агентом
- **📢 Всей команде** — broadcast всем агентам
- **⚙️ GitHub** — просмотр кода, коммитов, деплоя

## 🚨 Важные ограничения

- Данные агентов хранятся в `/tmp/` — **сбрасываются при рестарте** Railway
- Для постоянного хранения подключи **Railway PostgreSQL** или используй внешнюю БД
- LLaMA через Groq ограничен контекстом ~8000 токенов — очень длинный код может не поместиться

## 💡 Советы

- Агент пишет код в блоке ```python``` — бот автоматически предложит применить его
- Через ⚙️ GitHub → 🛠 Патч вручную — можно описать задачу словами, инженер сам напишет код
- Все ответы агентов дублируются в групповой чат (если настроен `GROUP_CHAT_ID`)
